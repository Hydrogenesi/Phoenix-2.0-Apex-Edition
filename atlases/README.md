## Atlas Philosophy

Traditional documentation describes systems linearly—text flows from beginning to end, specifications list requirements sequentially. Atlases transcend linearity by presenting multiple dimensions simultaneously. A single diagram can reveal structural hierarchy, data flow, temporal evolution, and conceptual relationships in unified visual form.

## The Six Atlases

**Recursion Atlas**: Maps self-similar patterns across scales—how operators contain operators, families contain families, systems contain systems. Visualizes the fractal nature of Phoenix architecture.

**Operator Flow**: Traces execution pathways through the Triadic architecture. Shows how data transforms as it passes from Substrate through Universal to Apex, revealing the operational choreography.

**Triad Morphology**: Documents transformation patterns—how structures morph as they move between layers, how operators evolve through elevation, how canonical forms emerge from primitive foundations.

**Substrate Geometry**: Detailed geometric documentation of foundational forms. Provides construction specifications for all primitive shapes, patterns, and sigils.

**Codex Hierarchy**: The master map—complete system structure from highest Apex concepts to deepest Substrate primitives, showing all relationships and dependencies.

**Apex Convergence**: Illustrates how recursive operations converge toward canonical forms, visualizing the descent toward perfect optimization.
