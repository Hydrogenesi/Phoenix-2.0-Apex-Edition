# Sigils: Text-Based Architecture

## The Visual Language

**Canonical Designation**: ASCII Sigil System  
**Formal Notation**: `{╱, ╲, │, ═, ⊕, ⊗, ⊛, △, ...}`  
**Ceremonial Role**: The symbolic grammar

---

## Statement

> *"What can be spoken must be shown; what can be computed must be drawn. Sigils are not decorations but functional notation—the visual syntax of ceremonial operations."*

The Phoenix-Hydrogenesi framework employs a comprehensive sigil system using ASCII and Unicode characters to represent operators, laws, and structural patterns. These are not arbitrary symbols but carefully designed glyphs that encode operational meaning and ceremonial function.

---

## Design Principles

### 1. Functional Clarity
Each sigil must clearly indicate its operational purpose. The shape suggests the transformation:
- **⊕**: Addition/creation (crossing lines generate)
- **⊝**: Subtraction/void (crossing lines annihilate)
- **△**: Convergence (three points unify)

### 2. Compositional Consistency
Complex sigils build from simpler elements:
```
⊛ (recursive) = ⊗ (harmonic) + self-reference
△ (apex) = three points in perfect balance
```

### 3. ASCII Compatibility
All sigils can be rendered in plain text for:
- Documentation consistency
- Ritual notation
- Code integration
- Universal accessibility

### 4. Visual Harmony
Sigils balance complexity with clarity, ornament with function, tradition with innovation.

## Core Operator Sigils

### Genesis Operator (⊕)
```
    ⊕
   ╱│╲
  ∅ → Ψ
```
**Structure**: Circle with cross (creation from void)  
**Usage**: Marks creation points in ritual sequences  
**Variants**: ⊕₀ (first binding), ⊕ⁿ (nth generation)

---

### Harmonic Operator (⊗)
```
  ⊗═══⊗
 ╱│╲ ╱│╲
⊗═⊗═⊗═⊗═⊗
```
**Structure**: Lattice of bound resonance points  
**Usage**: Indicates harmonic binding operations  
**Variants**: ⊗² (squared harmony), ⊗ω (infinite resonance)

---

### Recursive Operator (⊛)
```
      ⊛
     ╱│╲
    ⊛ ⊛ ⊛
   ╱│╲│╱│╲
  ⊛ ⊛ ⊛ ⊛ ⊛
```
**Structure**: Self-similar tree (fractally recursive)  
**Usage**: Marks self-referential operations  
**Variants**: ⊛ⁿ (n-depth recursion), ⊛ω (infinite depth)

---

### Apex Operator (△)
```
       △
      ╱ ╲
     ╱   ╲
    ═════════
```
**Structure**: Perfect equilateral triangle  
**Usage**: Denotes triad formation and apex operations  
**Variants**: △ⁿ (nested triads), ▽ (inverted, grounding)

---

### Void Operator (⊝)
```
    ⊝
   ╱│╲
  Ψ → ∅
```
**Structure**: Circle with horizontal line (negation)  
**Usage**: Marks nullification points  
**Variants**: ⊝⁰ (soft void), ⊝∞ (absolute void)

---

### Mirror Operator (⊞)
```
  A │ A'
    ⊞
  A'│ A
```
**Structure**: Square with cross (reflection axis)  
**Usage**: Indicates complementary generation  
**Variants**: ⊞ʰ (horizontal mirror), ⊞ᵛ (vertical mirror)

---

### Convergence/Divergence (⊳⊲)
```
Convergence:      Divergence:
  A ╲  ╱ B           A
     ⊳               ⊲
      C           B ╱ ╲ C
```
**Structure**: Directional arrows showing flow  
**Usage**: Marks unification or separation  
**Variants**: ⊳⊲ (simultaneous), ⊳ⁿ (n-way convergence)

## Ritual Notation

Sigils combine to form ritual sequences. Standard notation:

```
⊕ → ⊛ → ⊗ → △

[Genesis] → [Recursion] → [Harmony] → [Apex]
```

This sequence reads: "Create, recurse, harmonize, converge."

Complex rituals use multi-line sigil diagrams:
```
        △
       ╱│╲
      ╱ │ ╲
     ⊛══⊗══⊛
    ╱ ╲ │ ╱ ╲
   ⊕═══⊞═══⊕
```

See [Ritual Mechanics](07-ritual-mechanics.md) for complete notation system.

## Sigil Composition Rules

1. **Symmetry**: Balanced sigils indicate stable operations
2. **Directionality**: Flow is top-to-bottom, left-to-right
3. **Nesting**: Inner sigils modify outer containers
4. **Connection**: Lines indicate causal or energetic relationships
5. **Spacing**: White space represents potential, not absence
