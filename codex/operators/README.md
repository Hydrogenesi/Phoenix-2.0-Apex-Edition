## Creating New META Operators

Future META operators must follow these guidelines:

### Required Components
- **Harmonic Table**: Frequency, phase, amplitude, stratum
- **Ceremonial Definition**: Mathematical notation
- **Core Properties**: Primary role and capabilities
- **Invariant Specification**: What invariants are maintained
- **Violation Detection**: How violations are detected
- **Reversibility Properties**: Can operations be reversed?
- **Sigil**: Visual representation
- **Invocation Words**: Formal invocation language
- **Activation Ceremony**: How to activate
- **Integration Patterns**: How it integrates with engines

### Naming Convention
- Use "META_" prefix for new META operators
- Use descriptive names (e.g., META_VERIFY, META_BALANCE)
- Maintain consistency with existing META operator naming

### Authority Requirements
- All META operators operate at Crown Level (Stratum IV)
- Must not conflict with existing META operator authorities
- Special authorities (like Terminal Law) require explicit justification
