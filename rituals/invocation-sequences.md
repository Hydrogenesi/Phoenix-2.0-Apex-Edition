## Overview

Invocation sequences define the precise order and manner in which the [Eight Operators](../operators/README.md) must be applied to achieve specific transformational goals. Each sequence represents a tested pathway through operator space, optimized for safety, efficiency, and coherence with the [Universal Laws](../laws/universal/).

## Fundamental Invocations

### Single Operator Invocations

#### Genesis Invocation (⊕)
```
Preparation: Empty substrate, clear intention
Invocation: ⊕(void) → entity
Validation: Entity existence confirmed
```

**Purpose**: Manifest new pattern from void  
**Duration**: Instantaneous  
**Prerequisites**: None  
**Risks**: Minimal (most stable operation)

---

#### Harmonic Invocation (⊗)
```
Preparation: Two compatible entities (A, B)
Invocation: A ⊗ B → bound_pair
Validation: Resonance detected, binding confirmed
```

**Purpose**: Create resonant binding between entities  
**Duration**: 1-3 cycles  
**Prerequisites**: Entities must share frequency space  
**Risks**: Dissonance if incompatible frequencies

---

#### Recursive Invocation (⊛)
```
Preparation: Single entity with self-reference capacity
Invocation: ⊛(entity) → recursive_entity
Validation: Self-similarity confirmed across scales
```

**Purpose**: Establish recursive identity  
**Duration**: Variable (depends on recursion depth)  
**Prerequisites**: Entity must be recursion-compatible  
**Risks**: Infinite loops if depth limit not set

---

#### Apex Invocation (△)
```
Preparation: Three balanced entities (A, B, C)
Invocation: △(A, B, C) → apex_triad
Validation: Tri-column balance achieved
```

**Purpose**: Form unified apex structure from triad  
**Duration**: 5-10 cycles (convergence time)  
**Prerequisites**: Three entities, balanced attributes  
**Risks**: Instability if balance not maintained

---

#### Void Invocation (⊝)
```
Preparation: Target entity, void anchors established
Invocation: ⊝(entity) → trace
Validation: Entity dissolved, trace archived
```

**Purpose**: Dissolve entity while preserving trace  
**Duration**: 2-5 cycles  
**Prerequisites**: ⊕ anchor must exist  
**Risks**: **HIGH** - Can cascade if uncontrolled

---

#### Mirror Invocation (⊞)
```
Preparation: Source entity, reflection plane defined
Invocation: ⊞(entity) → reflected_entity
Validation: Symmetry verified, both entities stable
```

**Purpose**: Create reflection/inversion of entity  
**Duration**: Instantaneous  
**Prerequisites**: Clear symmetry axis  
**Risks**: Moderate (dual management required)

---

#### Convergence Invocation (⊳)
```
Preparation: Multiple related entities
Invocation: ⊳(entity₁, entity₂, ..., entityₙ) → unified
Validation: Single coherent entity, all attributes preserved
```

**Purpose**: Merge multiple entities into unified whole  
**Duration**: Variable (n/2 cycles for n entities)  
**Prerequisites**: Entities must be convergence-compatible  
**Risks**: Information loss if incompatible

---

#### Divergence Invocation (⊲)
```
Preparation: Composite entity, separation criteria
Invocation: ⊲(composite) → (part₁, part₂, ..., partₙ)
Validation: All parts independent, sum equals whole
```

**Purpose**: Separate composite into constituent parts  
**Duration**: Variable  
**Prerequisites**: Entity must have separable structure  
**Risks**: Essence loss if improperly partitioned
