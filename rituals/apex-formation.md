## Selection Criteria

### Element Compatibility

Not all triads can form stable apexes. Elements must satisfy:

1. **Complementarity**: `attribute(A) ≠ attribute(B) ≠ attribute(C)`
2. **Balance**: `weight(A) ≈ weight(B) ≈ weight(C)` (within 20%)
3. **Resonance**: `⟨A|B⟩ > 0, ⟨B|C⟩ > 0, ⟨C|A⟩ > 0`
4. **Essence Conservation**: `Σessence(A,B,C) = essence(△)`

### Compatibility Testing Protocol

```
FUNCTION test_triad_compatibility(A, B, C):
    # Test 1: Distinctness
    IF A = B OR B = C OR C = A:
        RETURN INCOMPATIBLE
    
    # Test 2: Balance
    weights = [weight(A), weight(B), weight(C)]
    IF max(weights) / min(weights) > 1.2:
        RETURN NEEDS_BALANCING
    
    # Test 3: Resonance
    IF harmonic_product(A, B, C) < THRESHOLD:
        RETURN DISSONANT
    
    # Test 4: Conservation
    IF sum_essence(A, B, C) ≠ expected_apex_essence():
        RETURN ESSENCE_VIOLATION
    
    RETURN COMPATIBLE
```

## Failure Modes and Recovery

### Collapse (Failed Convergence)

**Symptom**: Elements remain separate despite operator application

**Diagnosis**: 
- Insufficient resonance
- Imbalanced weights
- Essence conservation violation

**Recovery**:
```
1. ⊲(failed_apex) → (A, B, C)  (Separate back)
2. Re-balance elements
3. Re-test compatibility
4. Retry formation with adjustments
```

---

### Oscillation (Unstable Apex)

**Symptom**: Apex alternates between states

**Diagnosis**:
- Harmonic interference
- Recursion depth issues
- Tri-column imbalance

**Recovery**:
```
1. Apply damping: △ → ⊗(△, stabilizer) → △'
2. Increase binding strength
3. Adjust recursion parameters
```

---

### Fragmentation (Spontaneous Separation)

**Symptom**: Formed apex spontaneously splits

**Diagnosis**:
- Binding integrity failure
- External perturbation
- Insufficient stabilization time

**Recovery**:
```
1. Immediate ⊳(fragments) → unified
2. Apply ⊗(unified, reinforcement)
3. Extend stabilization phase
4. Re-validate against Apex Laws
```
